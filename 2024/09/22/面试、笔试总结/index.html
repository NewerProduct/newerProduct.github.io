HTTP状态码

以下是HTTP状态码的主要类别及一些常见的状态码：

1xx：信息性状态码

  • 100 Continue：初始部分已收到，客户端可以继续发送请求的其余部分。
  • 101 Switching Protocols：服务器已理解客户端的请求，并将使用不同的协议来处理请求。

2xx：成功状态码

  • 200 OK：请求已成功处理。
  • 201 Created：请求已成功处理，并创建了新的资源。
  • 204 No Content：请求成功处理，但不返回内容。

3xx：重定向状态码

  • 301 Moved Permanently：请求的资源已永久移动到新位置。
  • 302 Found：请求的资源临时移动到另一个URL。
  • 304 Not Modified：请求的资源未改变，客户端可以使用缓存的版本。

4xx：客户端错误状态码

  • 400 Bad Request：服务器无法理解请求，可能是因为语法错误。
  • 401 Unauthorized：请求未授权，需要用户身份验证。
  • 403 Forbidden：服务器理解请求，但拒绝执行。
  • 404 Not Found：请求的资源未找到。

5xx：服务器错误状态码

  • 500 Internal Server Error：服务器遇到意外情况，无法完成请求。
  • 502 Bad Gateway：作为网关或代理的服务器收到无效响应。
  • 503 Service Unavailable：服务器暂时不可用，通常是过载或维护。

parseInt函数

parseInt 是一个用于将字符串解析为整数的 JavaScript 函数。它通常用于从文本输入中提取数字。

基本语法

  • string: 必需。要被解析的字符串。
  • radix: 可选。一个介于 2 和 36 之间的整数，表示基数（在其他进制下表示数字的地方）。例如，10 表示十进制，16 表示十六进制。

返回值

如果字符串开头是一个有效的数字，则返回该数字（以整数形式）；如果解析失败，则返回 NaN（Not-a-Number）。

注意事项

  1. 自动忽略前导空格: parseInt 会忽略字符串开头的所有空格。
  2. 非数字字符的处理: parseInt 只会解析字符串中最开始的数字字符，如果遇到第一个非数字字符就停止解析。

除以0的规则

在 JavaScript 中，除以零的运算遵循 IEEE 754 浮点数标准，具体规则如下：

  • 任意正数 除以 零 的结果是 正无穷大 (Infinity)。
  • 任意负数 除以 零 的结果是 负无穷大 (-Infinity)。
  • 零 除以 零 的结果是 NaN（Not-a-Number）。

如何创建一个BFC

什么是BFC：BFC（Block Formatting Context）是一个独立的渲染区域，块级元素在这个区域中进行独立的布局，形成一个隔离的环境。BFC 用于控制元素的布局和内容流动。它有助于解决一些常见的布局问题，比如清除浮动、避免元素重叠等。

如何创建BFC

在 CSS 中，以下条件会创建 BFC：

  1. 根元素：<html> 元素。
  2. 浮动元素：任何 CSS float 属性值不为 none 的元素。
  3. 绝对定位元素：任何 CSS position 属性值为 absolute 或 fixed 的元素。
  4. 内联块元素：应用了 display: flow-root; 的元素。
  5. 表格元素：如 <table>、<table-cell>、<table-caption> 等。
  6. overflow 属性：应用了 overflow 属性且值不为 visible 的任何元素（如 overflow: auto; 或 overflow: hidden;）。

4. Math.pow()

答案:

答案:

  • Math.trunc(4.9) 返回 4（去掉小数部分）。
  • Math.trunc(-4.9) 返回 -4（去掉小数部分）。

ES5、ES6类的概念区别

ECMAScript 5（ES5）和 ECMAScript 6（ES6，也称为 ES2015）都引入了对“类”的支持，但它们的实现方式和理念有所不同。以下是 ES5 和 ES6 中类的主要概念和区别：

1. ES5 中的类

在 ES5 中，没有正式的类语法。相反，类的实现是基于构造函数和原型的方法。

定义一个类（构造函数）

在 ES5 中，创建“类”的方式是使用构造函数和原型：

2. ES6 中的类

ES6 引入了更为通用和简洁的 class 语法，使得面向对象编程的代码更加清晰、易读。

定义一个类

在 ES6 中，使用 class 关键字来定义类，使用 constructor 方法来定义构造函数，并且可以直接在类中定义方法。

3. 主要区别

3.1. 语法

  • ES5：使用构造函数和原型链。
  • ES6：使用 class 关键字，语法更加清晰。

3.2. 原型继承

  • ES5：继承通常通过设置原型对象。

  • ES6：使用 extends 关键字实现类的继承，使代码更简洁。

3.3. 静态方法

  • ES5：没有专门的语法定义静态方法，只能通过直接在构造函数上添加方法来实现。

  • ES6：可以使用 static 关键字轻松定义静态方法。

3.4. Getter 和 Setter

  • ES5：不支持使用简洁语法定义 getter 和 setter。

  • ES6：可以使用 get 和 set 关键字定义。

4. 总结

  • ES5 中的类是通过构造函数和原型实现的，而 ES6 引入了更直观的 class 语法，增强了面向对象编程的表达力。
  • ES6 大大简化了类和继承的定义，使得代码更加清晰、易于理解。
  • ES6 还提供了 Getter 和 Setter，以及静态方法等新特性，使得定义类的能力更为丰富。

浏览器的盒模型

标准盒模型

设置box-sizing:content-box默认值

标准盒模型遵循 W3C 的 CSS 规范，是现代浏览器（如 Chrome、Firefox、Safari 和 IE 11 及更新版本）默认的盒模型处理方式。该模型的计算方式如下：

  • 元素的实际宽度和高度由 内容区域的宽度和高度（width 和 height 属性）组成，不包括内边距（padding）、边框（border）和外边距（margin）。
  • 公式：
    • 总宽度 = width + padding-left + padding-right + border-left + border-right + margin-left + margin-right
    • 总高度 = height + padding-top + padding-bottom + border-top + border-bottom + margin-top + margin-bottom

怪异盒模型

设置box-sizing:border-box就可以使用怪异盒模型了

怪异盒模型主要用于老版本的浏览器（特别是 IE5 和 IE6），在这些浏览器中，width 和 height 属性的计算方式与标准盒模型不同。具体来说，width 和 height 属性的值包括内边距和边框。

  • 公式：
    • 总宽度 = width + padding + border + margin
    • 总高度 = height + padding + border + margin

Vue部分

couputed和watch的区别

Computed 计算属性

  • 缓存：计算属性是基于它们的依赖进行缓存的。只有当计算属性所依赖的数据发生变化时，它才会重新计算。这意味着如果依赖没有变化，多次访问计算属性会立即返回之前计算的结果，而不需要再次执行函数。
  • 依赖性：计算属性是声明式的，它的值是依赖于其他数据自动计算出来的。当依赖的数据发生改变时，计算属性的值也会随之更新。
  • 使用场景：当你需要根据其他数据动态地计算出一个值，并且这个值会被多次用到时，使用计算属性会更高效。例如，过滤列表或计算总和等。

Watch 监听器

  • 无缓存：监听器不会缓存结果，每次触发都会执行指定的回调函数。
  • 灵活性：监听器可以执行异步操作或者开销较大的操作，比如发起网络请求、修改多个状态等。
  • 使用场景：当你需要在数据变化时执行一些复杂的逻辑，或者需要执行异步操作时，应该使用监听器。例如，当某个值变化时向服务器发送数据更新。

总结

  • Computed 更适合用于简单的数据处理和衍生计算，这些计算是基于现有的数据并且希望利用Vue的响应式系统来自动管理这些计算。
  • Watch 则更适合处理数据变化后需要执行的一些复杂逻辑，尤其是那些涉及到异步操作或多个状态更改的情况。

Vue2和Vue3中响应式定义有什么区别？

主要区别

  1. API 设计
    • Vue 2：使用 data 选项，必须是一个函数，返回一个对象。
    • Vue 3：使用 setup 函数和 Composition API，提供了 ref 和 reactive 两种方式来定义响应式数据。
  2. 响应式机制
    • Vue 2：基于 Object.defineProperty，只能劫持已存在的属性，无法监听新属性的添加。
    • Vue 3：基于 Proxy，可以监听对象属性的增删和数组的变化。
  3. 灵活性和复用性
    • Vue 2：选项式 API（Options API）将逻辑分散在不同的选项中（如 data、methods、computed 等），适合简单的组件。
    • Vue 3：Composition API 提供了更灵活的逻辑组织方式，可以将相关的逻辑封装在一起，便于复用和测试。
  4. TypeScript 支持
    • Vue 2：对 TypeScript 的支持相对较弱，需要额外的配置和类型声明。
    • Vue 3：对 TypeScript 的支持更好，提供了更好的类型推断和类型安全。

总结

Vue 3 通过引入 Composition API 和 ref、reactive 等新特性，提供了更灵活、更强大的响应式数据定义方式。这些改进不仅提高了代码的可维护性和复用性，还增强了对 TypeScript 的支持，使得 Vue 3 成为了一个更加现代化和高效的前端框架。

CSS选择器有哪些？优先级排序？

CSS选择器是用于选取文档中的元素并应用样式规则的关键工具。下面是一些常见的CSS选择器类型及其优先级排序（从高到低）：

CSS选择器类型

  1. 内联样式 (style 属性) - 直接在HTML标签中定义的样式，如 <div style="color: red;">Hello</div>。
  2. ID选择器 (#id) - 通过元素的ID来选择元素，如 #header { color: blue; }。
  3. 类选择器 (.class) - 通过元素的类名来选择元素，如 .error { color: red; }。
  4. 属性选择器 ([attr]) - 通过元素的属性来选择元素，如 [type="text"] { border: 1px solid black; }。
  5. 伪类选择器 (:pseudo-class) - 用于选择处于特定状态下的元素，如 a:hover { color: green; }。
  6. 伪元素选择器 (::pseudo-element) - 用于选择元素的特定部分，如 p::first-line { font-weight: bold; }。
  7. 标签选择器 (element) - 通过元素名称来选择元素，如 div { margin: 10px; }。
  8. 通用选择器 (*) - 选择所有元素，如 * { box-sizing: border-box; }。
  9. 子选择器 (>) - 选择作为某个元素直接子元素的所有元素，如 ul > li { list-style: none; }。
  10. 相邻兄弟选择器 (+) - 选择紧接在另一个元素后的元素，前提是两个元素拥有相同的父元素，如 h1 + p { margin-top: 0; }。
  11. 一般兄弟选择器 (~) - 选择前一个元素之后的所有兄弟元素，如 h1 ~ p { color: gray; }。
  12. 组合选择器 - 将多个选择器组合在一起，如 .container .item, .container .highlight { background-color: yellow; }。

优先级排序

CSS选择器的优先级计算遵循以下规则：

  • 内联样式：1000
  • ID选择器：100
  • 类选择器、属性选择器、伪类选择器：10
  • 标签选择器、伪元素选择器：1
  • 通用选择器、组合选择器：0

优先级是累加的，即一个选择器的最终优先级是其内部各个部分优先级的总和。例如，选择器 .class #id element 的优先级为 111（10 + 100 + 1）。

当多个规则应用于同一个元素时，具有更高优先级的选择器将覆盖优先级较低的选择器。如果优先级相同，则最后定义的规则生效。

相对定位和绝对定位他们都是相对于谁进行定位的？

在CSS中，相对定位（position: relative;）和绝对定位（position: absolute;）都是用于控制元素在页面上的位置，但它们的定位基准点不同。下面是它们各自的定位基准点：

相对定位（position: relative;）

  • 定位基准点：相对定位的元素是相对于其正常位置进行定位的。也就是说，元素的位置会基于它在标准文档流中的位置进行偏移。
  • 影响布局：相对定位的元素仍然占据原来的空间，不会影响其他元素的布局。
  • 示例： 在这个例子中，.relative 元素会从它原本的位置向下移动10像素，向右移动20像素，但原来的布局空间仍然保留。

绝对定位（position: absolute;）

  • 定位基准点：绝对定位的元素是相对于最近的非 static 定位的祖先元素进行定位的。如果没有任何这样的祖先元素，则相对于初始包含块（通常是浏览器窗口）进行定位。
  • 影响布局：绝对定位的元素会脱离标准文档流，不再占据原来的空间，因此不会影响其他元素的布局。
  • 示例： 在这个例子中，.child 元素会相对于 .parent 元素进行定位，从 .parent 的左上角向下移动10像素，向右移动20像素。

关键点总结

  • 相对定位：相对于自身正常位置进行定位，仍然占据原来的空间。
  • 绝对定位：相对于最近的非 static 定位的祖先元素进行定位，脱离标准文档流，不占据原来的空间。

理解这些定位方式的基准点和行为，可以帮助你更有效地控制页面布局和元素位置。

  2. 通过input标签指定type为file来实现上传，如需接受多个上传文件，可指定属性multiple来上传多个文件，只是这种上传方式要求用户一次选择多个文件，而不能一次选择一个。

你的项目有提到将返回的图片展示在页面上，这个过程是怎么实现的呢？

当我们将图片上传，并按照自己的要求准备点击生成图片时，此时会进行发送Axios发送网络请求来请求接口，我们会获得返回包含有处理过后图片的base64格式的文件，我们只需要将该base64指定为img标签的src属性即可让图片成功显示在前端页面。
